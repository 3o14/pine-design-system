# Switch

설정이나 기능을 켜고 끄는 토글 컨트롤입니다.
[Base UI Switch](https://base-ui.com/components/react-switch/)를 기반으로 접근성을 기본으로 지원합니다.

Checkbox와 달리 Switch는 변경이 즉각적으로 반영될 때 사용합니다. 저장 버튼 없이 바로 설정을 적용하는 경우에 적합합니다.

## Props

### `checked`

Switch의 켜짐/꺼짐 상태를 제어합니다. 이 prop을 사용하면 controlled 컴포넌트가 됩니다.

- **타입**: `boolean`
- **기본값**: `undefined`

### `defaultChecked`

Switch의 초기 상태를 설정합니다. uncontrolled 방식으로 사용할 때 씁니다.

- **타입**: `boolean`
- **기본값**: `undefined`

### `onCheckedChange`

Switch 상태가 변경될 때 호출되는 콜백 함수입니다.

- **타입**: `(checked: boolean) => void`

### `intent`

Switch 트랙의 색상 의도를 결정합니다. 기능이 켜진 상태일 때 표시되는 색상에 영향을 줍니다.

| 값 | 기본값 | 설명 |
| --- | :---: | --- |
| `primary` | ✓ | 일반적인 설정 토글에 사용합니다. |
| `secondary` | | 보조 설정에 사용합니다. |
| `success` | | 활성화가 긍정적 의미를 가질 때 사용합니다. |
| `warning` | | 활성화에 주의가 필요할 때 사용합니다. |
| `danger` | | 활성화가 위험한 동작을 나타낼 때 사용합니다. |
| `neutral` | | 중립적인 토글에 사용합니다. |

### `size`

Switch의 크기를 결정합니다.

| 값 | 기본값 | 설명 |
| --- | :---: | --- |
| `small` | | 밀도 높은 설정 목록에 사용합니다. |
| `medium` | ✓ | 일반적인 설정 화면에 사용합니다. |
| `large` | | 충분한 클릭 영역이 필요할 때 사용합니다. |
| `xlarge` | | 모바일 환경처럼 큰 터치 영역이 필요할 때 사용합니다. |

### `label`

Switch 옆에 표시되는 라벨 텍스트입니다. 라벨 클릭 시 Switch가 토글됩니다.

- **타입**: `string`

### `disabled`

Switch를 비활성화합니다. 비활성화 상태에서는 상태를 변경할 수 없습니다.

- **타입**: `boolean`
- **기본값**: `false`

### `readOnly`

Switch를 읽기 전용으로 만듭니다.

- **타입**: `boolean`
- **기본값**: `false`

## 사용 가이드

### Switch vs Checkbox

| 상황 | 권장 컴포넌트 |
| --- | --- |
| 켜짐/꺼짐 즉시 반영 (저장 불필요) | **Switch** |
| 폼 제출 시 최종 반영 | **Checkbox** |
| 여러 항목 중 다수 선택 | **Checkbox** |
| 단일 설정 토글 | **Switch** |

### 접근성

`label` prop을 통해 스크린리더가 Switch의 목적을 읽을 수 있도록 반드시 레이블을 제공하세요.

## 코드

```tsx
import { Switch } from "pine-design-system";

// 기본 사용
<Switch label="알림 받기" defaultChecked />

// Controlled
<Switch
  label="다크 모드"
  intent="neutral"
  checked={isDark}
  onCheckedChange={setIsDark}
/>

// 비활성화
<Switch label="관리자만 변경 가능" disabled defaultChecked />
```
