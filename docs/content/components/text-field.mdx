# TextField

텍스트를 입력받는 폼 컴포넌트입니다.
[Base UI Field](https://base-ui.com/components/react-field/)와 [Base UI Input](https://base-ui.com/components/react-input/)을 기반으로 라벨, 입력 영역, 헬퍼 텍스트를 단일 컴포넌트로 제공합니다. `multiline` prop으로 `<textarea>`로 전환할 수 있습니다.

## Props

### `label`

입력 필드 위에 표시되는 라벨 텍스트입니다. 라벨은 `<label>` 요소로 렌더링되어 입력 필드와 자동으로 연결됩니다.

- **타입**: `string`

### `placeholder`

값이 없을 때 표시되는 플레이스홀더 텍스트입니다.

- **타입**: `string`

### `helperText`

입력 필드 아래에 표시되는 보조 텍스트입니다. 에러 메시지, 힌트, 형식 안내 등에 사용합니다.

- **타입**: `string`

### `variant`

입력 필드의 시각적 스타일을 결정합니다.

| 값 | 기본값 | 설명 |
| --- | :---: | --- |
| `outline` | ✓ | 테두리가 있는 일반적인 입력 스타일입니다. |
| `filled` | | 배경이 채워진 입력 스타일입니다. 다른 폼 요소와 구분이 필요할 때 사용합니다. |

<ComponentPreview>
  <TextFieldVariantDemo />
</ComponentPreview>

### `status`

입력 필드의 유효성 상태를 표시합니다. `status`에 따라 테두리 색상과 헬퍼 텍스트 색상이 변경됩니다.

| 값 | 기본값 | 설명 |
| --- | :---: | --- |
| `default` | ✓ | 기본 상태입니다. |
| `error` | | 유효성 검사 실패 상태입니다. 헬퍼 텍스트를 오류 메시지로 활용하세요. |
| `success` | | 유효성 검사 성공 상태입니다. 헬퍼 텍스트를 확인 메시지로 활용하세요. |

<ComponentPreview>
  <TextFieldStatusDemo />
</ComponentPreview>

### `size`

입력 필드의 크기를 결정합니다.

| 값 | 기본값 | 설명 |
| --- | :---: | --- |
| `small` | | 밀도 높은 폼에 사용합니다. |
| `medium` | ✓ | 일반적인 폼에서 사용합니다. |
| `large` | | 충분한 입력 영역이 필요할 때 사용합니다. |
| `xlarge` | | 큰 입력 영역이 필요할 때 사용합니다. |

### `rounded`

모서리 곡률을 결정합니다.

| 값 | 기본값 | 설명 |
| --- | :---: | --- |
| `small` | | 작은 모서리 반경 |
| `medium` | ✓ | 기본 모서리 반경 |
| `large` | | 큰 모서리 반경 |

### `multiline`

`true`로 설정하면 `<input>` 대신 `<textarea>`로 렌더링됩니다.
`rows` prop으로 기본 표시 행 수를 지정할 수 있습니다.

- **타입**: `boolean`
- **기본값**: `false`

<ComponentPreview>
  <TextFieldMultilineDemo />
</ComponentPreview>

### `rows`

`multiline`이 `true`일 때 textarea의 기본 높이(행 수)를 지정합니다.

- **타입**: `number`
- **기본값**: `3`
- **조건**: `multiline: true`일 때만 사용 가능

### `fullWidth`

`true`로 설정하면 부모 컨테이너의 전체 너비를 채웁니다.

- **타입**: `boolean`
- **기본값**: `false`

### `disabled`

입력 필드를 비활성화합니다.

- **타입**: `boolean`
- **기본값**: `false`

### `required`

폼 유효성 검사 시 필수 입력 항목으로 표시합니다.

- **타입**: `boolean`
- **기본값**: `false`

## 사용 가이드

### 에러 상태 표시하기

`status="error"`와 `helperText`를 함께 사용해 구체적인 오류 원인을 안내하세요. 단순히 빨간 테두리만 표시하는 것보다 사용자가 무엇을 수정해야 하는지 명확하게 알 수 있습니다.

```tsx
<TextField
  label="이메일"
  status="error"
  helperText="올바른 이메일 형식이 아닙니다."
  value={email}
  onChange={(e) => setEmail(e.target.value)}
/>
```

## 코드

```tsx
import { TextField } from "pine-design-system";

// 기본 사용
<TextField label="이름" placeholder="홍길동" />

// 유효성 상태
<TextField
  label="이메일"
  type="email"
  status="error"
  helperText="올바른 이메일 형식을 입력하세요."
/>

// 여러 줄 입력 (textarea)
<TextField
  label="자기소개"
  multiline
  rows={5}
  placeholder="간단하게 소개해 주세요."
/>

// 전체 너비
<TextField label="주소" fullWidth />
```
