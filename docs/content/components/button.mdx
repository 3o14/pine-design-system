# Button

사용자가 명확한 액션을 수행할 수 있도록 돕는 인터랙션 컴포넌트입니다.
[Base UI Button](https://base-ui.com/components/react-button/)을 기반으로 키보드 접근성과 포커스 관리를 기본으로 지원합니다.

## 예제

### Variant

버튼의 시각적 강조 수준을 결정합니다. 화면 내 액션의 중요도에 맞게 선택하세요.

| 값 | 기본값 | 설명 |
| --- | :---: | --- |
| `solid` | ✓ | 배경색이 채워진 스타일. 가장 강조가 필요한 주요 CTA에 사용합니다. |
| `outline` | | 테두리만 있는 스타일. Solid보다 낮은 강조의 보조 액션에 사용합니다. |
| `ghost` | | 배경과 테두리가 없는 스타일. 가장 낮은 강조가 필요할 때 사용합니다. |
| `weak` | | 반투명 배경 스타일. Solid와 함께 보조 액션으로 조합하기에 적합합니다. |

### Size

버튼의 크기를 결정합니다. 화면 맥락과 클릭 영역 필요도에 맞게 선택하세요.

| 값 | 기본값 | 설명 |
| --- | :---: | --- |
| `small` | | 좁은 공간이나 보조 액션에 사용합니다. |
| `medium` | ✓ | 가장 범용적인 크기입니다. |
| `large` | | 주요 CTA 또는 충분한 클릭 영역이 필요할 때 사용합니다. |
| `xlarge` | | 큰 화면의 주목도 높은 CTA에 사용합니다. |

### Intent

버튼의 의미적 목적을 색상으로 표현합니다. 동작의 성격에 맞는 intent를 선택하세요.

| 값 | 기본값 | 설명 |
| --- | :---: | --- |
| `primary` | ✓ | 앱의 핵심 기능에 사용하는 주요 액션입니다. |
| `secondary` | | Primary와 함께 조합하는 보조 액션입니다. |
| `success` | | 완료, 저장, 확인 등 긍정적 결과의 액션에 사용합니다. |
| `warning` | | 주의가 필요한 액션에 사용합니다. |
| `danger` | | 삭제, 초기화 등 되돌릴 수 없는 액션에 사용합니다. 사용자에게 위험을 명확히 알려주세요. |
| `neutral` | | 의미적 강조 없이 중립적인 액션에 사용합니다. |

## 사용 가이드

### Variant 선택하기

화면 내 액션의 중요도에 따라 적절한 variant를 선택합니다. 한 화면에 `solid` 버튼은 하나만 사용하는 것을 권장합니다. `solid`가 여러 개면 사용자가 어떤 액션이 주요 액션인지 판단하기 어렵습니다.

- `solid` → 한 화면에서 가장 중요한 단일 CTA
- `weak` → Solid와 함께 쓰는 보조 액션
- `outline` / `ghost` → 낮은 강조가 필요한 부가 액션

### Intent 선택하기

버튼의 동작이 가지는 의미적 성격을 `intent`로 표현합니다. 특히 삭제·초기화처럼 되돌릴 수 없는 액션에는 반드시 `danger`를 사용해 사용자에게 명확히 경고하세요.

## 코드

```tsx
import { Button } from "pine-design-system";

// 기본 사용
<Button>시작하기</Button>

// 위험한 액션
<Button intent="danger" variant="outline">삭제하기</Button>

// 전체 너비 CTA
<Button fullWidth size="large">확인</Button>
```
